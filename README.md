# django_sprint4

Проект сайта-блога на Django с пользователями, публикациями и комментариями.

---

## Требования

- **Python 3.9.x**
- `pip`
- встроенный `venv`
- ОС: Windows / Linux / macOS

> Проект **не гарантирует** корректную работу на Python 3.10+ / 3.12 (из-за версии Pillow).

---

### 1. Клонирование репозитория

```bash
git clone git@github.com:fianchettoo/django_sprint4.git
cd <ваш_путь_к_проекту>
```

### 2. Создание виртуального окружения

Windows
```bash
python -m venv venv
```

Linux / macOS
```bash
python3 -m venv venv
```

### 3. Активация виртуального окружения

Windows
```bash
source venv\Scripts\activate
```

Linux / macOS
```bash
source venv/bin/activate
```

### 4. Обновление pip

```bash
python -m pip install --upgrade pip
```

### 5. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 6. Инициализация базы данных

```bash
cd blogicum
python manage.py migrate
```

### 7. Запуск сервера

```bash
python manage.py runserver
```

## Работа с приложением
### Пользователи
- **Регистрация**

    - Перейдите на страницу регистрации;

    - Заполните форму и создайте аккаунт.

- **Авторизация**

    - Войдите под созданным пользователем.

    - После входа становятся доступны:

        - создание публикаций;

        - комментарии;

        - редактирование своего профиля.

---

### Публикации
- **Создание публикации**

    - Доступно только авторизованным пользователям;

    - Перейдите на страницу добавления публикации.

Заполните:

    1) заголовок;

    2) текст;

    3) дату публикации;

    4) категорию и местоположение;

    5) изображение (опционально).

 Нажмите «Отправить».

- **Редактирование публикации**

Доступно только автору публикации.

Кнопка редактирования отображается:

- в профиле пользователя;

- на странице публикации.

Чужие публикации редактировать нельзя.

- **Удаление публикации**

Доступно только автору.

После удаления происходит редирект в профиль пользователя.

- **Скрытие публикации**

Поле is_published отвечает за видимость публикации.

Если публикация снята с публикации:

 - не видна другим пользователям;

 - доступна автору в его профиле.

Будущие публикации отображаются только после наступления даты публикации.

---

### Комментарии
- **Добавление комментария**

Доступно только авторизованным пользователям.

Комментарий появляется под публикацией.

- **Редактирование комментария**

Доступно:

- автору комментария;

- администратору.

- **Удаление комментария**

Доступно:

- автору комментария;

- администратору.

---

### Профиль пользователя

Страница профиля доступна по адресу:

    /profile/<username>/


На странице отображается:

- информация о пользователе;

- список его публикаций;

- количество комментариев у каждой публикации.

Дополнительно (только для владельца профиля):

- ссылка на редактирование профиля;

- ссылка на смену пароля.
